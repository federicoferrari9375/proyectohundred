# Primer Proyecto.
## 1. Herramientas Instaladas

- Cursor IDE: Editor de código basado en inteligencia artificial que facilita el desarrollo asistido.
- GitHub: Plataforma utilizada para el control de versiones y almacenamiento de repositorios.
- Codex: Extensión utilizada para asistir en la generación y análisis de código dentro del editor.


## 2. Pasos Completados

1. Creación de una cuenta de usuario en GitHub.
2. Instalación del editor Cursor IDE.
3. Configuración inicial del editor.
4. Instalación y activación del complemento Codex dentro de Cursor.
5. Exploración de las herramientas de inteligencia artificial disponibles dentro del editor.
6. Sincronización del editor Cursor con la cuenta de GitHub para permitir el control de versiones.

## 3. Problemas Encontrados y Soluciones

### 3.1 Instalación del complemento Claude Code

-Problema: 
Al intentar instalar el complemento Claude Code, no se encontraba disponible específicamente para el editor Cursor, ya que aparecía únicamente como extensión para Visual Studio Code. Además, no se mostraba la opción para iniciar sesión.

-Solución: Se investigó la documentación y el funcionamiento interno del editor Cursor, descubriendo que el soporte para Claude ya se encuentra integrado dentro del propio editor.  

Dentro del chat del editor se desactivó la opción de selección automática de modelos de IA y se pudieron visualizar las distintas opciones disponibles, entre ellas:

- Claude Opus 4.6
- Claude Sonnet 4.6

Esto confirmó que Claude ya estaba preestablecido en Cursor, por lo que no era necesario instalar una extensión adicional.

### 3.2 Sincronización de Cursor con GitHub

-Problema: Se presentó un inconveniente al intentar sincronizar el editor Cursor con la cuenta de GitHub.

Solución: Se realizó una búsqueda de información para identificar el procedimiento correcto de autenticación. Posteriormente se utilizaron los códigos de autorización proporcionados por GitHub, lo que permitió completar exitosamente la sincronización entre Cursor y el repositorio remoto.

## 4. Estado Actual

El entorno de desarrollo quedó correctamente configurado, permitiendo:

- Utilizar Cursor IDE como editor principal.
- Acceder a diferentes modelos de IA integrados dentro del editor.
- Sincronizar proyectos con GitHub para control de versiones.
